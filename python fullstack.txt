Rest API?
REST API: REST 아키텍처의 제약조건을 준수하는 애플리케이션 프로그래밍 인터페이스.
REST: Representational State Transfer의 줄임말.

API: Application Programming Interface는 애플리케이션 소프트웨어를 구축하고
통합하는 정의 및 프로토콜 세트.
API는 정보 제공자와 정보 사용자 간의 계약으로 지칭되며 소비자에게 필요한 콘텐츠와 생산자에게
필요한 콘텐츠를 구성한다.

[웹 서비스 개발 동향]
1세대: 정적 웹페이지: 다른 컴퓨터에 위치한 정보를 웹 브라우저를 통해 볼 수 있는 기능.
2세대: 틀은 동일하지만 들어가있는 데이터가 다른 형태: 동적 홈페이지:
데이터 베이스에다 데이터를 넣고 HTML틀을 만든 다음에 거기다 로직을 사용해서
데이터를 적합하게 끼워넣는 프로그램의 필요가 생겼다. C->PHP

3세대:웹페이지가 수시로 바뀌는 경우.
해당코드를 수시로 바꾸지 말고 특정 부분만 바꾸도록 구조화 시킬 필요가 생겼다.
MVC패턴: Model- View- Control 패턴으로 구조화된 프레임워크 사용
보여지는 부분, 요청에 따른 데이터 처리 부분, 다뤄지는 데이터 담당 부분.

4세대: 빠르기
꼭 웹페이지만 가져오는 게 아니라, 원하는 데이터를 가져올 수도 있다: 세부분을 각자 다른 컴퓨터에.
Rest API통해 컴퓨터들 사이에 데이터를 요청, 응답할 수 있다.
